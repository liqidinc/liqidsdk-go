Liqid™ SDK Programming Reference for Go - Version 3.2.0

Categories:

CategoryDescription
BackupRelates to backing up the Director configuration
ClientFunctions related to the state of the client struct/object, or to its state.
This includes logging in and out, and any settings for SDK logging.
CoordinatesThese functions relate to internal API mechanisms, and are not particularly useful for the SDK client.
They are presented here for completeness, and due to the possibility that one or another of them might eventually be needed.
DeviceFunction generally related to devices
DeviceDiscoveryFunctions relating to discovery of PCI devices
DeviceInfoFunctions which retrieve miscellaneous device information
DeviceStatusFunctions which retrieve the status of various devices
FabricThese functions relate to the full configuration (fabric) which is managed by the currently-connected Director.
GroupIncludes functions related to the configuration of a group.
GroupDeviceRelatorFunctions which move devices into and out of group free pools
GroupPoolFunctions which manage edit mode for the group pool
MachineGeneral functions related to configured machines
MachineDeviceRelatorFunctions which attach devices to and detach them from machines
ManagerThese functions manage the managed-entity descriptors which are used for discovering devices.
MessageDigestThese functions relate to secure authentication of REST API function calls.
They are not of use to the SDK clients, as the SDK provides wrapper methods for these functions.
See commentary on authentication models and functions related to logging in and logging out.
NodeStatusRetrieves NodeStatus entities describing the various nodes
PowerManagementManages power states for a node
SshIncludes functions which get and set SSH status
SystemFunctions which control system power
VersionIncludes functions related to reporting software component version information.

GroupDescribes a configured group which contains a free device pool and some number of configured machines.
This struct does not contain information regarding the related entities; that information must be obtained via other functions/methods.
GroupComputeDeviceRelatorDescribes a relation between a group and a compute device

Does not contain relations with devices; that information resides in the various device relators.

GroupPoolDescribes a group pool
GroupStorageDeviceRelatorDescribes a relation between a group and a storage device
HardwareComponentDescribes the hardware details of a component
LiqidClientThis struct is necessary for managing all communication with the Liqid Director.
For OO languages, it serves as the class on which all methods are defined.
For non-OO languages, it is a required parameter for all function invocations.

MachineDescribes a configured machine
